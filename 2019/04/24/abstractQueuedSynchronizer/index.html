AbstractQueuedSynchronizer

Posted on 2019-04-24 | Edited on 2019-05-06 | Views:

AbstractQueuedSynchronizer 主要做了三件事：获取锁，释放锁，先进先出队列

JAVA-API注释：提供一个框架，用于实现依赖先进先出（FIFO）等待队列的阻塞锁和相关同步器（信号量，事件等）
如果要使用这个类作为同步器的基础，可以使用getState、setState、compareAndSetState读写同步状态，根据需要重新定义以下方法:
 –tryAcquire 独占模式获取锁
 –tryRelease 独占模式释放锁
 –tryAcquireShared 共享模式获取锁
 –tryReleaseShared 共享模式释放锁
 –isHeldExclusively 当前线程是否持有锁

入队

入队操作干了四件事情：

  1. 封装线程Node： 将当前线程封装成Node同时指定独占或共享模式
  2. CAS初始化头尾节点： 当未初始化时，初始化头节点head和尾节点tail
  3. 队尾追加： 根据FIFO原则，新增节点会被追加到队尾
  4. CAS更新tail： 通常将新增节点作为新的tail，然后返回新的tail

独占模式

只有一个线程能够持有锁，其他线程只能等待.需要实现一下三个方法

  1. tryAcquire
  2. tryRelease
  3. isHeldExclusively
    获取锁的方式有三种：
  4. acquire: 不响应中断获取锁
  5. acquireInterruptibly: 响应中断获取锁
  6. tryAcquireNanos: 响应中断和超时获取锁
    释放锁的方式一种： release
    状态变化：0 -> -1 -> 0

共享模式

多个线程能持有锁，需要实现一下两个方法

  1. tryAcquireShared
  2. tryReleaseShared
    获取锁的方式有三种：
  3. acquireShared: 不响应中断获取锁
  4. acquireSharedInterruptibly: 响应中断获取锁
  5. tryAcquireSharedNanos: 响应中断和超时获取锁
    释放锁的方式一种： releaseShared.

除了获取资源和释放资源两个操作，共享锁的实现思路几乎和独占锁一样
