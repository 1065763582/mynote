状态

线程的状态定义在Thread内部的枚举类里， 共六个

转换流程如下图：

start

start方法是启动一个新的线程来执行，首先判断Thread的状态是否为New它的主要逻辑是通过native方法star0实现的，Java虚拟机将会调用这个线程的run方法， 而run方法是在当前线程执行.

sleep

sleep使当前线程进入等待状态，需要注意的是当前现在并不会释放锁

join

当使用一个线程对象调用join方法，这个方法会挂起当前线程，直到调用这个方法的线程对象执行完成后才会继续执行当前线程。
join方法上有synchronized标识，线程对象调用该方法就获取到了线程对象到锁，内部使用Object.wait等待，那么在没有其他线程显式调用该对象的notify或notifyAll时，是如何唤醒当前线程的？ 有JVM释放锁， 当线程对象里的逻辑执行完成后，JVM会自动调用线程对象的notifyAll
